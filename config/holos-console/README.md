# holos-console config tree

Kustomize bases and overlays for the `holos-console` binary's
`templates.holos.run` API surface.

## Directory layout

```
config/holos-console/
├── crd/          # Generated CRDs — templates.holos.run API group.
│                 # Regenerated by `make manifests`; never hand-edited.
├── admission/    # Hand-authored CEL ValidatingAdmissionPolicy +
│                 # ValidatingAdmissionPolicyBinding manifests (scope-escape
│                 # and cross-tenant guards).
├── rbac/         # ClusterRole for holos-console-templates plus binding.
│                 # role.yaml is regenerated by `make manifests`; the
│                 # ServiceAccount and ClusterRoleBinding are hand-authored.
├── namespace-scoped/ # Overlay for the holos-console ServiceAccount.
└── cluster-scoped/  # Overlay that composes crd/ + admission/ + rbac/.
                     # Apply with: kubectl apply -k config/holos-console/cluster-scoped/
```

## Apply order

```sh
# 1. Namespace-scoped resources first (ServiceAccount must exist before
#    ClusterRoleBindings bind it).
kubectl apply -k config/namespace-scoped/

# 2. Cluster-scoped resources (CRDs, VAPs, ClusterRole).
kubectl apply -k config/cluster-scoped/
```

The `config/cluster-scoped/` overlay includes both `holos-console` and
`holos-secret-injector` cluster-wide resources. See the top-level
`config/` overlays for the consolidated apply paths used in production.

For the holos-secret-injector equivalent, see
`config/secret-injector/namespace-scoped/README.md`.
